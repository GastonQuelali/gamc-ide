Decisión de Base de Datos

Para una plataforma catastral con datos geoespaciales, relaciones complejas y requerimiento de integridad, la elección es relacional con extensión geoespacial. Los datos NoSQL juegan un rol complementario.

✓ Recomendado Principal

PostgreSQL + PostGIS

Motor relacional open source con la mejor extensión geoespacial del mercado. Estándar de la industria GIS.
Soporte nativo de geometrías (puntos, polígonos, líneas)
Consultas espaciales SQL estándar (ST_Within, ST_Intersects…)
Compatible con ArcGIS Server vía SDE
Transacciones ACID — crítico para catastro
Gratuito y sin licencias
Requiere DBA para tunning en alta concurrencia

SQL Server (Existente)

Mantener para datos legados de recaudación e integración con sistemas actuales del municipio.
Ya instalado y licenciado
Integración con sistemas Windows municipales
Soporte espacial con SQL Server Spatial
Licencias costosas para escalar
Menor soporte GIS vs PostGIS
Rol: solo datos de recaudación legacy

Redis (NoSQL Cache)

No reemplaza el relacional. Se usa como capa de caché y manejo de sesiones para soportar 5K usuarios.
Caché de tiles y respuestas de mapas
Sesiones de usuario en tiempo real
Cola de trabajos async (con Celery)
No persiste datos críticos
Solo complementario al relacional

Seguridad y Acceso
🔐
Autenticación
JWT + OAuth2. Roles: ciudadano, inspector, admin, ejecutivo
🌐
SSL/TLS
HTTPS obligatorio. Certificados Let's Encrypt o institucional
🛡️
WAF
NGINX + ModSecurity. Protección contra SQLi, XSS, DoS
⏱️
Rate Limiting
100 req/min ciudadano, 1000 req/min interno. Redis backed
🗂️
Datos sensibles
Propietarios y valuaciones solo autenticados. Datos públicos anonimizados
💾
Backups
PostgreSQL: backup diario automático. SQL Server: política existente
Resumen de Decisiones Clave

⚡ Decisiones Arquitectónicas Finales

Decisión Elección Alternativa descartada Razón
Base de datos principal PostgreSQL + PostGIS MongoDB Soporte geoespacial nativo, ACID, integración ArcGIS
Backend API FastAPI (Python) Node.js / .NET Ecosistema GIS/ML en Python, ArcPy compatible
Contenerización Docker Compose Kubernetes Escala adecuada, menor complejidad operativa
Frontend React + ArcGIS SDK Angular / Vue Comunidad, componentes ArcGIS nativos para React
App móvil campo ArcGIS Field Maps React Native custom Offline nativo, sincronización Esri sin desarrollo
Proxy / Web server NGINX Apache Performance, reverse proxy, WAF con ModSecurity
Datos SQL Server existente Mantener (legacy) Migrar todo Minimizar riesgo, integrar vía ETL con Airflow
Caché Redis Memcached Soporta sesiones, colas (Celery), pub/sub
